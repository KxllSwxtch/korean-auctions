# 🔧 Руководство по исправлению проблем с Glovis сессией

## 📋 Описание проблемы

В логах наблюдаются следующие ошибки:

```
WARNING | ⚠️ Проблемы с сессией Glovis: JavaScript редирект на логин
WARNING | ⚠️ Сессия невалидна, обновляем...
ERROR   | ❌ Не удалось восстановить валидную сессию
```

## ✅ Реализованные исправления

### 1. **Обновление cookies**

- Обновлены cookies в `glovis_service.py` на основе рабочего примера из `glovis-curl-request.py`
- Исправлен `JSESSIONID` на актуальный

### 2. **Экстренный механизм восстановления**

Добавлены новые методы в `GlovisService`:

- `_load_cookies_from_curl_file()` - автоматическая загрузка cookies из curl файла
- `_emergency_cookies_update()` - экстренное обновление при полном отказе сессии

### 3. **Улучшенная обработка ошибок**

- Многоуровневая система восстановления сессии
- Автоматическая попытка экстренного обновления при стандартных методах
- Детализированное логирование каждого этапа

### 4. **Автоматический мониторинг**

Создан `glovis_session_auto_updater.py` для:

- Непрерывного мониторинга состояния сессии
- Автоматического обновления cookies при проблемах
- Сохранения актуальных данных в кэш

## 🚀 Использование

### Базовое использование (без изменений)

```python
from app.services.glovis_service import GlovisService

service = GlovisService()
cars = await service.get_car_list()
```

### Ручное обновление cookies

```python
# Если нужно принудительно обновить cookies
if not await service.ensure_valid_session():
    # Экстренное обновление из curl файла
    service._emergency_cookies_update()
```

### Автоматический мониторинг

```bash
# Одноразовая проверка
python glovis_session_auto_updater.py

# Непрерывный мониторинг (каждые 5 минут)
python glovis_session_auto_updater.py --continuous

# Мониторинг с интервалом 2 минуты
python glovis_session_auto_updater.py --continuous --interval 120
```

## 🔄 Алгоритм восстановления сессии

1. **Проверка валидности** - стандартная проверка сессии
2. **Стандартное обновление** - обновление через `refresh_session()`
3. **Экстренное обновление** - загрузка cookies из `glovis-curl-request.py`
4. **Повторная проверка** - валидация после каждого шага

```
Запрос -> Проверка сессии -> Стандартное обновление -> Экстренное обновление -> Результат
    ↓            ↓                     ↓                        ↓
   OK        Невалидна            Не помогло              Последняя попытка
    ↓            ↓                     ↓                        ↓
Продолжить   Обновить             Экстренное             Ошибка/Успех
```

### Источники cookies (по приоритету):

3. **Дефолтные cookies** - встроенные в сервис

## 📋 Логирование

Все действия логируются с детализированной информацией:

- `✅` - Успешные операции
- `⚠️` - Предупреждения (проблемы с сессией)
- `❌` - Ошибки (критические проблемы)
- `🔄` - Операции обновления
- `🚨` - Экстренное восстановление

```
2025-06-19 16:08:54 | SUCCESS | ✅ Экстренное обновление cookies помогло!
```

## 🛠 Устранение неисправностей

### Если экстренное обновление не помогает:

1. **Проверьте актуальность curl файла**:

   - Убедитесь что `glovis-curl-request.py` содержит свежие cookies
   - JSESSIONID должен быть действующим

2. **Обновите curl файл**:

   - Зайдите на сайт в браузере
   - Скопируйте новые cookies из DevTools
   - Обновите `glovis-curl-request.py`

3. **Перезапустите сервис**:
   ```bash
   # Остановите и запустите заново
   kill $(pgrep -f "fastapi")
   python main.py
   ```

### Если проблемы повторяются:

- Запустите автоматический мониторинг: `python glovis_session_auto_updater.py -c`
- Установите более частую проверку: `--interval 60`
- Проверьте блокировку по IP
