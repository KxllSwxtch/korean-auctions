## 🔍 Проверка валидности сессии

### API Endpoints

#### 1. Проверка сессии

```bash
curl -X GET "http://localhost:8000/api/v1/glovis/check-session"
```

**Ответ при валидной сессии:**

```json
{
  "success": true,
  "message": "Сессия валидна",
  "data": {
    "is_valid": true,
    "status_code": 200,
    "response_size": 368876,
    "cookies_info": {
      "JSESSIONID": {
        "value": "rfrLu3sj9IRm4MoFMcfD...",
        "domain": "",
        "path": "/",
        "expires": null
      }
    },
    "issues": [],
    "redirect_location": null
  }
}
```

**Ответ при проблемах с сессией:**

```json
{
  "success": false,
  "message": "Сессия требует обновления",
  "data": {
    "is_valid": false,
    "status_code": 302,
    "issues": ["Редирект на страницу логина", "HTTP 401 - Неавторизован"]
  }
}
```

#### 2. Обновление сессии

```bash
curl -X POST "http://localhost:8000/api/v1/glovis/refresh-session"
```

## 🤖 Автоматические механизмы

### 1. **Предварительная проверка**

API автоматически проверяет валидность сессии перед каждым запросом к Glovis:

```python
# Проверяем валидность сессии перед запросом
session_check = await self.check_session_validity()
if not session_check.get('is_valid', False):
    logger.warning("⚠️ Сессия невалидна, обновляем...")
    self.refresh_session()
```

### 2. **Обработка ошибок в реальном времени**

При получении ответов с ошибками аутентификации:

```python
if '<script>location.href=\'/login.do\';</script>' in response.text:
    logger.error("❌ Получен JavaScript редирект на логин, сессия истекла")
    # Автоматическое восстановление
    self.refresh_session()
```

### 3. **HTTP 401 обработка**

```python
if response.status_code == 401:
    logger.info("🔄 Ошибка 401, обновляем сессию...")
    self.refresh_session()
```

## 🚨 Индикаторы проблем с сессией

### 1. **HTTP коды ошибок**

- `401 Unauthorized` - Сессия истекла
- `302 Found` + `Location: /login.do` - Редирект на логин
- `403 Forbidden` - Доступ запрещен

### 2. **Содержимое ответа**

- `<script>location.href='/login.do';</script>` - JavaScript редирект
- Форма логина в коротком ответе (< 1000 символов)
- Текст "login" в коротком ответе

### 3. **Размер ответа**

- Нормальная страница: > 300,000 символов
- Страница ошибки: < 10,000 символов

## 🛠️ Устранение проблем

### Проблема: Сессия постоянно истекает

**Решение:**

1. Проверьте актуальность cookies в `app/services/glovis_service.py`
2. Обновите `JSESSIONID` из рабочего браузера
3. Проверьте номер аукциона (`atn`) и другие параметры

### Проблема: API возвращает пустые данные

**Решение:**

```bash
# 1. Проверьте сессию
curl -X GET "http://localhost:8000/api/v1/glovis/check-session"

# 2. Обновите сессию при необходимости
curl -X POST "http://localhost:8000/api/v1/glovis/refresh-session"

# 3. Повторите запрос
curl -X GET "http://localhost:8000/api/v1/glovis/cars"
```

### Проблема: Ошибки парсинга

**Решение:**

1. Убедитесь, что получаете HTML, а не страницу логина
2. Проверьте изменения в структуре сайта Glovis
3. Обновите селекторы в парсере
